# PIN-156: Landing Page Identity Guidelines

**Status:** ACTIVE
**Category:** Marketing / Positioning / Phase A
**Created:** 2025-12-24
**Related PINs:** PIN-128, PIN-154, PIN-155

---

## Executive Summary

This PIN codifies the language and positioning for AOS's public-facing identity during Phase A (Learning Acceleration Era). The core principle:

> **"Open Control Plane (Foundational Access)" - not "Free Forever"**

Identity anchoring decisions made now determine pricing power later.

---

## The Three Traps to Avoid

### Trap 1: "Free" as Identity

| Dangerous | Safe |
|-----------|------|
| "AOS is free" | "AOS offers open access" |
| "Free AI governance" | "Foundational AI governance" |
| "No cost to start" | "Open to builders" |

**Why it matters:** When "free" becomes your identity, every pricing discussion feels like betrayal. Customers think "you changed" rather than "you graduated."

### Trap 2: "Forever" as Promise

| Dangerous | Safe |
|-----------|------|
| "Free forever" | "Foundational access" |
| "Always free tier" | "Open control plane" |
| "Permanent free access" | "Builder access" |

**Why it matters:** "Forever" is a legal and emotional trap. It forces you to honor a promise made when you had zero data about what's valuable.

### Trap 3: "Stability" as Guarantee

| Dangerous | Safe |
|-----------|------|
| "Production-ready" | "Exploratory access" |
| "Enterprise-grade" | "Building towards enterprise" |
| "99.9% SLA" | "Improving reliability daily" |

**Why it matters:** Promising stability in Phase A means you'll spend engineering time maintaining features before you know if they matter.

## Approved Terminology

### Primary Identity

```
"Open Control Plane (Foundational Access)"
```

### Word-by-Word Justification

| Word | Function | Protects Against |
|------|----------|------------------|
| **Open** | Signals accessibility | "Free" baggage |
| **Control Plane** | Technical framing | Attracts builders, not freeloaders |
| **Foundational** | Implies layers above | Expectation of forever-free |
| **Access** | Temporary permission | Permanent right entitlement |

### Approved Phrases

| Context | Use This | Not This |
|---------|----------|----------|
| Pricing | "Foundational access" | "Free tier" |
| Features | "Core capabilities" | "Free features" |
| Limits | "Foundational limits" | "Free tier limits" |
| Upgrades | "Authority tiers" | "Paid plans" |
| Users | "Builders" | "Free users" |
| Entry | "Open access" | "Free sign-up" |

## FAQ Language

### "Is AOS free?"

**DO:**
```
AOS offers foundational access through our Open Control Plane.
This gives builders full access to core governance capabilities
while we develop authority tiers for teams needing advanced features.
```

**DON'T:**
```
Yes! AOS is completely free forever. We'll never charge for
the features you use today.
```

### "Will you start charging?"

**DO:**
```
We're building authority tiers for teams needing advanced
governance, longer retention, and priority support. Foundational
access will continue to serve builders exploring AI governance.
```

**DON'T:**
```
No, we promise the free tier will always remain exactly as it is
today with no limits or restrictions.
```

### "What's the catch?"

**DO:**
```
No catch. We're in our learning phase, building alongside early
adopters. Your usage helps us understand what governance features
matter most. Authority tiers will come when we've earned the
right to charge for them.
```

**DON'T:**
```
There's no catch! It's free forever with no limits. We make
money from enterprise customers so individuals never pay.
```

## Phase Transition Language

### Phase A (Now): Learning Acceleration

```
"Open Control Plane - build and learn with us"
"Foundational access for builders"
"Exploring AI governance together"
```

### Phase B (Future): Authority Pricing

```
"Foundational access remains, Authority tiers now available"
"Graduate to Authority when you're ready"
"Core capabilities stay open, advanced governance for operators"
```

The transition should feel like **graduation**, not **betrayal**.

## Checklist for New Content

Before publishing any public content, verify:

- [ ] No use of "free" as identity anchor
- [ ] No "forever" promises
- [ ] No stability guarantees beyond current state
- [ ] Uses "Foundational" or "Open" instead of "Free"
- [ ] Uses "Authority" instead of "Paid/Premium"
- [ ] Sets expectation of evolution
- [ ] Invites partnership, not entitlement

---

## Why This Matters

### The Math

```
"Free forever" positioning:
- 10,000 users expect free forever
- 50% feel betrayed when pricing arrives
- 5,000 angry users, negative reviews, churn

"Foundational access" positioning:
- 10,000 users expect graduation path
- 80% understand authority tiers are coming
- 8,000 potential graduates, positive sentiment
```

### The Psychology

Users who join understanding "foundational" naturally expect premium tiers.
Users who join expecting "free forever" feel cheated by the same offering.

The product is identical. The positioning determines the outcome.
